CAP. 01

Introdução ao
Ecossistema Flutter

Antes de escrever qualquer linha de código, é essencial entender o que é Flutter, qual é o papel do Dart, e por que essa combinação conquistou milhões de desenvolvedores no mundo inteiro.

O que é Flutter?

Flutter é um framework de código aberto criado pelo Google para construir aplicativos nativamente compilados para mobile (Android e iOS), web e desktop a partir de um único código-fonte.

Dart é a linguagem de programação por trás do Flutter. A distinção é fundamental: Flutter é a caixa de ferramentas (os pincéis, a tela, as tintas), enquanto Dart é o idioma com o qual você dá as instruções ao framework.

ConceitoDefiniçãoAnalogia
FlutterFramework de UI multiplataformaA caixa de ferramentas do artista
DartLinguagem de programaçãoO idioma que você fala
WidgetBloco de construção da UIOs tijolos da casa
pub.devRepositório de pacotesA loja de materiais

Por que Dart? Compilação AOT e JIT

Dart suporta dois modos de compilação que tornam o desenvolvimento extremamente eficiente:

⚡ JIT — Just-In-Time

  • Usado durante o desenvolvimento
  • Habilita o famoso Hot Reload
  • Alterou o código → vê na tela em <1s
  • Mantém o estado do app ao recarregar

🚀 AOT — Ahead-Of-Time

  • Usado na compilação final
  • Converte Dart em código nativo de máquina
  • Performance comparável a Swift/Kotlin
  • Apps rápidos e eficientes em produção
💡
Hot Reload é o superpoder do Flutter: altere o código e veja o resultado na tela instantaneamente, mantendo o estado do app. É como atualizar uma página web, só que para apps nativos!

CAP. 02

Fundamentos da
Linguagem Dart

Dart é uma linguagem fortemente tipada com inferência de tipos, suporte a programação orientada a objetos e recursos modernos como Null Safety. Vamos dominar os fundamentos que você vai usar todos os dias.

Variáveis e Tipagem

main.dart
// Tipagem explícita
String nome = 'Flutter';
int versao = 3;
double pi = 3.14159;
bool ativo = true;

// Inferência de tipo com 'var'
var linguagem = 'Dart';   // Dart infere: String
var ano = 2025;           // Dart infere: int

// Imutabilidade
final cidade = 'São Paulo';   // Definido em runtime, não muda
const PI = 3.14159;           // Constante em tempo de compilação
KeywordMutável?Quando usar
var✅ SimTipo inferido, pode ser reatribuído
String/int/double✅ SimTipagem explícita, mais legível
final❌ NãoAtribuído uma vez em runtime. Objetos não-nulos.
const❌ NãoConstante em tempo de compilação. Valores literais.

Null Safety: o fim dos crashes de null

null_safety.dart
// Sem Null Safety (❌ perigoso)
String nome;   // Poderia ser null → crash em runtime!

// COM Null Safety (✅ seguro)
String nome = 'Ana';       // Nunca pode ser null
String? apelido;           // Pode ser null (o '?' declara isso)

// Operador '?.' — acesso seguro (retorna null se null)
print(apelido?.length);    // Safe: retorna null se apelido for null

// Operador '!' — afirma que NÃO é null (use com certeza!)
print(apelido!.length);    // Força acesso; pode dar erro!

// Operador '??' — valor padrão se null
String exibir = apelido ?? 'sem apelido';
⚠️
Regra de ouro: Use ? quando o valor realmente PODE ser nulo. Use ! somente quando você tem certeza absoluta que não é nulo — caso contrário, você terá um erro em runtime.

Arrow Functions e Funções

functions.dart
// Função tradicional
int somar(int a, int b) {
  return a + b;
}

// Arrow Function (=>) — para uma única expressão
int somar(int a, int b) => a + b;

// Parâmetros nomeados e opcionais
void saudacao({required String nome, String titulo = 'Dev'}) {
  print('Olá, $titulo $nome!');
}
saudacao(nome: 'Ana', titulo: 'Sênior');

CAP. 03

A Filosofia dos
Widgets

No Flutter, tudo é widget. Textos, botões, imagens, layouts, animações e até o próprio app são widgets. Entender essa filosofia é a chave para pensar em Flutter.

A Widget Tree

Os widgets se organizam em uma hierarquia chamada Widget Tree. Cada widget pode ter filhos (children), formando uma árvore que representa toda a UI do app.

MaterialApp
└── Scaffold
├── AppBar
│ └── Text('Meu App')
└── Center
└── Column
├── Text('Olá!')
└── ElevatedButton

Stateless vs. Stateful

Esta é a dúvida mais comum entre iniciantes. A resposta é simples: tudo depende de se o widget precisa mudar durante a vida do app.

StatelessWidget

  • Imutável após criação
  • Não tem estado interno
  • Mais performático (sem rebuild desnecessário)
  • Ideal para: cards, textos, ícones fixos
  • Exemplo: um card de produto estático

StatefulWidget

  • Tem estado que pode mudar
  • Usa setState() para notificar mudanças
  • Flutter reconstrói o widget ao chamar setState
  • Ideal para: contadores, formulários, animações
  • Exemplo: botão de curtir, carrinho de compras

Widgets de Layout Essenciais

WidgetFunção
ScaffoldEstrutura base da tela. Fornece AppBar, body, FAB e Drawer.
AppBarBarra superior com título, ações e botão de navegação.
CenterCentraliza o filho horizontal e verticalmente.
ColumnEmpilha widgets verticalmente (um abaixo do outro).
RowAlinha widgets horizontalmente (lado a lado).
ContainerCaixa versátil: padding, margin, cor, decoração, tamanho.
ExpandedPreenche o espaço disponível dentro de Column/Row.
SizedBoxEspaço fixo entre widgets (como margem manual).

setState: dando vida à interface

counter.dart
class BotaoContador extends StatefulWidget {
  @override
  State<BotaoContador> createState() => _BotaoContadorState();
}

class _BotaoContadorState extends State<BotaoContador> {
  int _contagem = 0;

  void _incrementar() {
    setState(() {          // Notifica o Flutter para rebuild
      _contagem++;         // Atualiza o estado
    });
  }

  @override
  Widget build(BuildContext context) {
    return ElevatedButton(
      onPressed: _incrementar,
      child: Text('Cliques: $_contagem'),
    );
  }
}
ℹ️
Como funciona o setState: quando você chama setState(() { ... }), o Flutter marca o widget como "sujo" e chama o método build() novamente, reconstruindo a UI com os novos valores de estado.

CAP. 05

Projeto Prático:
Gerador de Frases Nerds

Vamos construir um app completo que une tudo que aprendemos: listas, estado, UI e interatividade. O Gerador de Frases Nerds exibe uma frase aleatória ao apertar um botão.

Estrutura do Projeto

main.dart — Parte 1: Setup
import 'dart:math';
import 'package:flutter/material.dart';

void main() => runApp(const MeuApp());

class MeuApp extends StatelessWidget {
  const MeuApp({super.key});
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Frases Nerds',
      theme: ThemeData(
        colorScheme: ColorScheme.fromSeed(
          seedColor: Colors.indigo,
          brightness: Brightness.dark,
        ),
      ),
      home: const GeradorFrases(),
    );
  }
}
main.dart — Parte 2: Lógica e UI
class GeradorFrases extends StatefulWidget {
  const GeradorFrases({super.key});
  @override
  State<GeradorFrases> createState() => _GeradorFrasesState();
}

class _GeradorFrasesState extends State<GeradorFrases> {
  final List<String> _frases = [
    'There are 10 types of people...',
    'It works on my machine!',
    'sudo make me a sandwich',
    'Dart is the best language!',
    '99 bugs in the code...',
  ];

  String _fraseAtual = 'Toque para gerar!';
  int _contador = 0;
  final _random = Random();

  void _gerarFrase() {
    setState(() {
      _fraseAtual = _frases[_random.nextInt(_frases.length)];
      _contador++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('🚀 Frases Nerds')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text('Geradas: $_contador'),
            const SizedBox(height: 24),
            Card(child: Padding(
              padding: const EdgeInsets.all(20),
              child: Text(_fraseAtual,
                style: const TextStyle(fontSize: 20),
                textAlign: TextAlign.center),
            )),
            const SizedBox(height: 32),
            ElevatedButton.icon(
              onPressed: _gerarFrase,
              icon: const Icon(Icons.refresh),
              label: const Text('Nova Frase!'),
            ),
          ],
        ),
      ),
    );
  }
}
